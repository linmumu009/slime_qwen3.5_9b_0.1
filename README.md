## 迭代记录规则

每次产生可提交的代码、配置或文档变更时，必须同时完成以下记录：

1. 在本 README 的“迭代摘要”表中新增一行，概括目标、核心变化与验证结果。
2. 在 `docs/iterations/` 中新建一份独立的完整迭代说明，历史文件不得覆盖。
3. 完整说明至少包含：迭代目标、变更内容、验证结果、风险与回滚方式、后续事项。
4. 文件名采用 `YYYY-MM-DD-NNN-short-title.md`，其中 `NNN` 为三位递增编号。

新迭代可复制 [`docs/iterations/TEMPLATE.md`](docs/iterations/TEMPLATE.md) 后填写。

- SSH 私钥必须保存在用户的 `.ssh` 目录中，不得放入或提交到本仓库。
